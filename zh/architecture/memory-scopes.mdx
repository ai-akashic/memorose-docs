# 记忆领域（Memory Domains）

当前 Memorose 模型使用三个主要领域：

- `agent`
- `user`
- `organization`

旧版文档描述的是以 `app` 为中心的层级结构。那已不再是主要的产品模型。

## 领域职责

### 智能体记忆（Agent Memory）

智能体记忆主要是过程性的。

适用于：

- 工具使用模式
- 执行痕迹（Execution Traces）
- 恢复策略
- 规划启发式（Planning Heuristics）
- 智能体特定的反思

### 用户记忆（User Memory）

用户记忆主要是事实性和偏好性的。

适用于：

- 身份事实
- 偏好
- 约束条件
- 长期个人上下文
- 绑定到单个用户的目标

### 组织记忆（Organization Memory）

组织记忆是可复用的共享知识。它应当从授权的源记忆中演化而来，而非将所有原始用户事件混入一个全局桶中。

适用于：

- 策略
- 术语
- 共享工作流
- 通用最佳实践
- 可复用的高阶洞察

## 原生与投射

- `agent` 和 `user` 记忆是原生领域（Native Domains）
- `organization` 记忆通常是从源记忆经过过滤、抽象和共享策略检查后投射（Projected）而来的

## 领域如何与 L0-L3 交互

层级和领域回答的是不同的问题：

- `L0-L3`：记忆如何随时间演化
- `agent/user/organization`：记忆属于谁，谁可以复用它

你需要同时具备这两个维度才能正确理解系统。
